# 回路 JSON ファイル仕様

回路を定義する JSON ファイルのフォーマットを規定します。

## 概要

回路 JSON ファイルはワイヤの一覧のみで構成されます。セル（ノード）はワイヤの端点から自動的に推論されるため、明示的なセル定義は不要です。

## スキーマ

### ルートオブジェクト

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `wires` | `Wire[]` | はい | ワイヤ定義の配列 |

### サブ回路対応フィールド（拡張）

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `modules` | `Module[]` | いいえ | サブ回路モジュールのインスタンス配列 |
| `subs` | `object` | いいえ | サブ回路定義のマップ（キー: サブ回路名） |
| `input` | `Input[]` | いいえ | 入力コンポーネント定義（generator） |
| `output` | `Output[]` | いいえ | 出力コンポーネント定義（tester） |

注: `generators` / `testers` は後方互換用として引き続き受理されます。

### Wire オブジェクト

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `src` | `[i32, i32]` | はい | 入力元セルの座標 `[x, y]` |
| `dst` | `[i32, i32]` | はい | 出力先セルの座標 `[x, y]` |
| `kind` | `string` | はい | ワイヤの極性。`"positive"` または `"negative"` |

### Module オブジェクト

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `string` | はい | 現在は `"sub"` のみ対応 |
| `sub_circuit` | `string` | はい | 参照するサブ回路名 |
| `input` | `[i32, i32][]` | はい | 親回路側の入力ポート座標 |
| `output` | `[i32, i32][]` | はい | 親回路側の出力ポート座標 |

### SubCircuit オブジェクト（`subs.<name>`）

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `wires` | `Wire[]` | はい | サブ回路内部のワイヤ |
| `sub_input` | `[i32, i32][]` | はい | サブ回路内部の入力ポート座標 |
| `sub_output` | `[i32, i32][]` | はい | サブ回路内部の出力ポート座標 |
| `modules` | `Module[]` | いいえ | ネストされたサブ回路モジュール |

### kind の値

| 値 | 説明 |
|---|---|
| `"positive"` | 入力元の値をそのまま伝搬する |
| `"negative"` | 入力元の値を反転して伝搬する |

## 制約

回路 JSON は以下の制約を満たす必要があります。違反した場合はエラーとなります。

1. **self-loop 禁止**: `src` と `dst` が同一座標であるワイヤは許可されません
2. **ワイヤペアの一意性**: 同一の `(src, dst)` を持つワイヤは 1 つのみ許可されます。`kind` に関わらず、ある 2 点間には最大 1 本のワイヤしか配置できません
3. **サブ回路参照の存在**: `modules[].sub_circuit` は `subs` に存在するキーでなければなりません
4. **サブ回路参照の循環禁止**: `subs` 間の参照は DAG である必要があります
5. **入出力数一致**: `modules[].input/output` の要素数は、参照先 `sub_input/sub_output` の要素数と一致する必要があります
6. **ポート列制約**: `input`, `output`, `sub_input`, `sub_output` はそれぞれ同一 x かつ連続 y の列を満たす必要があります

## サブ回路を含む例

```json
{
  "wires": [
    { "src": [0, 0], "dst": [2, 0], "kind": "positive" }
  ],
  "modules": [
    {
      "type": "sub",
      "sub_circuit": "inverter",
      "input": [[2, 0]],
      "output": [[3, 0]]
    }
  ],
  "subs": {
    "inverter": {
      "wires": [
        { "src": [0, 0], "dst": [1, 0], "kind": "negative" }
      ],
      "sub_input": [[0, 0]],
      "sub_output": [[1, 0]],
      "modules": []
    }
  }
}
```

## 最小の例

NOT ゲート（入力を反転する回路）:

```json
{
  "wires": [
    { "src": [0, 0], "dst": [1, 0], "kind": "negative" }
  ]
}
```

セル `(0, 0)` の値を反転してセル `(1, 0)` に伝搬します。セル `(0, 0)` と `(1, 0)` はワイヤ端点から自動的に生成されます。

## NAND ゲートの例

```json
{
  "wires": [
    { "src": [0, 0], "dst": [2, 0], "kind": "negative" },
    { "src": [1, 0], "dst": [2, 0], "kind": "negative" }
  ]
}
```

セル `(0, 0)` と `(1, 0)` を入力、`(2, 0)` を出力とする NAND ゲートです。2 本の Negative ワイヤの伝搬値が OR 合成されることで NAND 演算を実現します。

## 半加算器の例

```json
{
  "wires": [
    { "src": [0, 0], "dst": [1, 0], "kind": "positive" },
    { "src": [0, 1], "dst": [1, 0], "kind": "positive" },
    { "src": [0, 0], "dst": [1, 1], "kind": "negative" },
    { "src": [0, 1], "dst": [1, 1], "kind": "negative" },
    { "src": [1, 0], "dst": [2, 0], "kind": "negative" },
    { "src": [1, 1], "dst": [2, 0], "kind": "negative" },
    { "src": [2, 0], "dst": [3, 0], "kind": "negative" },
    { "src": [0, 0], "dst": [2, 1], "kind": "negative" },
    { "src": [0, 1], "dst": [2, 1], "kind": "negative" },
    { "src": [2, 1], "dst": [3, 1], "kind": "negative" }
  ]
}
```

入力セル `(0, 0)` と `(0, 1)` から、和 `sum` をセル `(3, 0)` に、桁上げ `carry` をセル `(3, 1)` に出力する半加算器です。

## シミュレーション出力

シミュレーション結果 JSON の `ticks` 配列には、各 tick 完了時点の状態が格納されます。

| フィールド | 型 | 説明 |
|---|---|---|
| `tick` | `u64` | 完了した tick の 0-based インデックス |
| `cells` | `object` | キーが `"x,y"`、値が `0` または `1` のセル状態 |

最初の tick 完了後の要素は `tick: 0` です。
